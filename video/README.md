# FlowHub Explainer Videos

Two short English explainer videos (end-user + technical) built with Remotion,
narrated by a local Piper voice (`en_US-amy-medium`), with a background music
bed and the FlowHub logo on the title cards. Isolated from the
.NET solution â€” its own Node toolchain. Final length is driven by the narration:
the committed `durations.json` reflects real Piper timing (~40s / ~44s) and is
regenerated by `just video-tts` whenever the scripts change.

## Quick start

From the repo root, two `just` recipes do everything:

```bash
just video-setup   # one-time: vendor Piper + voice model + ffmpeg, npm install
just video         # narration â†’ render both MP4s into video/out/
```

`just video` runs `video-setup` automatically if the toolchain is missing, so a
clean checkout only needs `just video`. Other recipes: `just video-tts`,
`just video-render`, `just video-clean`. See `just --list`.

## One-time setup (manual / what `video-setup` automates)

`just video-setup` vendors everything below under `video/tools/` (all git-ignored,
x86_64 Linux). To do it by hand or on another platform:

1. Install Node deps:
   ```bash
   cd video && npm install
   ```
2. Install [Piper](https://github.com/rhasspy/piper) and the English voice model
   `en_US-amy-medium`. Put the model files here:
   ```
   video/tools/voices/en_US-amy-medium.onnx
   video/tools/voices/en_US-amy-medium.onnx.json
   ```
   Or point `PIPER_BIN` / `PIPER_MODEL` env vars at your install.
3. Ensure `ffmpeg`/`ffprobe` are on PATH (the `just` recipes prepend the vendored
   `video/tools/ffmpeg/` instead).
4. Install a color-emoji font (e.g. Noto Color Emoji) so the Remotion renderer's
   Chromium draws emoji icons instead of empty squares. `video-setup` vendors it
   and activates it in your user font dir.
5. (Optional) Replace the silent `public/audio/music/bed.mp3` with a real
   royalty-free track and record attribution in `public/audio/music/LICENSE.md`.

## Editing content

- Narration lives in `scripts/*.en.md`. Each scene is delimited by
  `<!-- scene: <id> -->`. Scene ids must match keys in `durations.json` and the
  `audio/tts/<key>-<id>.wav` paths used by the compositions.
- Visuals live in `src/*.tsx` (`UserVideo`, `TechnicalVideo`) and `src/components/`.

## Generate narration + durations

```bash
npm run tts
```
Generates `public/audio/tts/*.wav` and overwrites `src/durations.json` so scene
timing matches the narration length.
